소켓 : 	네트워크에서 동작하는 프로그램의 도착지
	포트번호와 IP주소로 이루어져 있음
	서버와 클라이언트가 양방향 통신을 할 수 있게 도와주는 SW

통신절차 :
 1. 서버에서 서버용 소켓(ServerSocket)을 생성하고 클라이언트가 접속하길 기다림
 2. 클라이언트가 소켓(Socket)을 생성하여 서버로 연결을 요청
 3. 서버가 접속을 허가(accept)
 4. 서버와 클라이언트는 각각 통신을 위해 I/O 스트림을 생성
 5. 스트림을 통해 서버와 클라이언트가 통신(write<->read)
 6. 클라이언트가 모든 작업을 마친 후 클라이언트가 종료(close)
 7. 서버는 새로운 클라이언트의 접속을 위해 대기(accept) 하거나 종료(close)

웹소켓의 애너테이션
 @ServerEndPoint : 웹소켓 서버의 요청명을 설정
 @OnOpen : 클라이언트가 접속했을 때 요청되는 매서드 정의
 @OnMessage : 클라이언트로부터 메세지가 전송되었을 때 실행
 @OnClose : 클라이언트의 접속이 종료되면 실행되는 메서드 정의
 @OnError : 에러 발생시 실행되는 메서드 정의
