** 유효성 검사 :  사용자가 폼에서 입력한 데이터 값이 서버로 전송되기 전에 특정 규칙에 맞게 입력되었는지 검증하는 것
	예) 아이디 중복검사, 로그인 시 아이디 패스워드 검사, 회원가입시 입력값들, IP 패킷 검사

** 유효성 검사 작성시 반드시 점검해야 할 항목
	- 입력데이터가 null인지 확인
	- 날짜 또는 이메일 등은 형식에 맞는지 확인
	- 나이 등은 숫자인지 확인
	- 입력데이터의 제한 길이를 초과했는지 확인
	- 로그인할 때 아이디/패스워드 확인
	- 회원가입할 때 아이디 중복 여부 확인

** 유효성 검사의 종류
- 기본 유효성 검사
	: 사용자가 폼에 입력한 항목에 대하여 값이 존재하는지 확인
	: 데이터길이, 숫자 등 기본적인 것이 맞는지 확인

** 유효성 검사를 사용하는 방법으로 정규표현식(regex)을 사용함
정규표현식 : 특정한 규칙을 가진 문자열의 집합을 표현하는데 사용되는 형식 언어, 문자열의 특정 형태를 찾기 위해 패턴으로 표현한 수식
	장점 : 조건문을 매우 간단하게 표현 가능
	단점 : 코드가 간단한 만큼 가독성이 떨어짐
